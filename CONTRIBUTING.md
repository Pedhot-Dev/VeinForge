# Contributing to VeinForge

First off, thank you for considering contributing to VeinForge! It's people like you that make VeinForge such a great tool.

## How Can I Contribute?

### Reporting Bugs
- Use the **Bug Report** issue template.
- Provide as much detail as possible (logs, reproduction steps, etc.).

### Suggesting Enhancements
- Use the **Feature Request** issue template.
- Explain why this enhancement would be useful.

### Pull Requests
1. Fork the repository.
2. Create a new branch for your changes.
3. Ensure your code follows the existing style.
4. Add tests if applicable.
5. Submit a Pull Request with a clear description of your changes.

## Development Setup
- This project uses Gradle.
- Run `./gradlew build` to compile the project.
- Use an IDE like IntelliJ IDEA for the best experience.

## Style Guide
- Follow standard Java and Kotlin naming conventions.
- Keep methods concise and well-documented if they are complex.
